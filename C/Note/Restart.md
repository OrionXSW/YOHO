# 指针
C学到最后，就是在控制内存，指针作为C的灵魂，得好好学
**所有指针的大小都是固定的（比如 32 位系统里是 4 字节，64 位系统里是 8 字节）**

# 函数和数组
函数在内存里的情况和数组有相似之处，也有本质不同。
**数组是一段连续的内存，里面装的是一个个相同类型的元素。** 数组名在表达式中通常隐式转换为指向第一个元素的指针，也就是这段连续内存的起始地址。

**函数也是一段连续的内存，但里面装的是编译后的机器指令，不是数据。** 函数名代表的就是这段指令区域的**入口地址**。当你在代码里写下一个函数的名字，编译器会把它转换成指向这个入口的指针（函数指针）。这一点和数组名转换成首元素地址确实很像——都是拿到内存中某块连续区域的起始地址。

但它们有一个关键区别：  
数组名转换成指针后，指向的是“第一个元素”这种数据对象，你可以对它进行指针运算，比如加一、下标访问，去读或者写数组里的元素。  
函数名转换成函数指针后，指向的是“第一条指令”，你不能对它做加减运算，也不能通过它去读写指令本身（在大多数环境下，代码区是只读的，且标准不允许对函数指针做算术运算）。你唯一能做的就是通过这个指针去调用函数，让程序跳转到那个地址开始执行。

所以，你可以这样理解：

- 数组是一块连续的数据内存，数组名转为指向这块数据起点的指针。
- 函数是一块连续的代码内存，函数名转为指向这块代码起点的指针（函数入口地址）。

它们都是“名字隐式变成地址”，但数组地址拿来访问数据，函数地址拿来跳转执行。这就是函数在内存里的基本情况。

# typedef和define

## typedef
能给已有的类型名起别名，但必须加分号  ***(类型别名)***
`typedef 类型名 类型别名`
> **去掉 `typedef`，剩下的部分如果是一条合法的变量声明，那么原本写变量名的那个位置，就变成新的类型名。**

- 编译器**完全按语法位置判断**，没有任何语义猜测。
- 结构体、数组、函数指针等所有情况都遵循这同一条规则。

---
### 基本类型
```c
typedef unsigned char uc;
```
去掉 `typedef` → `unsigned char uc;`  
`uc` 是变量名 → 升级为类型别名，代表 `unsigned char`。

---
### 数组
```C
typedef int Arr[10];    // 定义别名：Arr 代表 int[10] 类型

Arr a;                  // 等价于 int a[10];
Arr b, c;               // 等价于 int b[10], c[10];

a[0] = 5;               // 和普通数组完全一样使用
```
去掉 `typedef` → `int Arr[10];`  
`Arr` 变成类型别名，代表 `int[10]` 数组。

---
### 函数指针
```c
typedef void (*FuncPtr)(int);
// 用这个别名声明函数指针变量
FuncPtr p;                       // p 是一个函数指针
FuncPtr q = NULL;                // 可以初始化为空
// 假设有一个符合签名的函数
void myCallback(int x) {
    printf("%d\n", x);
}
// 赋值：把函数名（即函数地址）赋给指针
p = myCallback;      // 也可以写 p = &myCallback，效果相同
// 调用：通过指针调用函数，就像直接调用一样
p(10);               // 输出 10
(*p)(20);            // 也可以这样，和上面等价
```
去掉 `typedef` → `void (*FuncPtr)(int);`  
`FuncPtr` 是函数指针变量名 → 变成类型别名，代表 `void (*)(int)`。

---

### 结构体

#### 1. 匿名结构体（最常用）
```c
typedef struct { int x; int y; } Point;
```
去掉 `typedef` → `struct { int x; int y; } Point;`  
`Point` 是变量名 → 变成类型别名，之后直接写 `Point p;`。

#### 2. 带标签的结构体（标签和别名可同名）
```c
typedef struct Node {
    int data;
    struct Node *next;
} Node;
```
去掉 `typedef` → `struct Node { ... } Node;`  
最后的 `Node` 是变量名 → 变成 `struct Node` 的别名。  
**结构体标签和普通标识符在不同名字空间，同名不冲突。**
##### 为什么可以在结构体内部定义指向该结构体的指针？

在 `typedef struct Node { ... struct Node *next; } Node;` 中，看起来我们还没定义完结构体，却已经在内部使用了 `struct Node *next`，这是合法的。原因是：

1. **标签提前可见**  
   当编译器读到 `struct Node {` 时，标签 `Node` 就已经在结构体标签的名字空间里被注册了（成为一个“不完全类型”）。此时 `struct Node` 这个类型名已经可以使用，哪怕大括号还没闭合。

2. **指针的大小是固定的**  
   声明一个指针变量，编译器只需要知道它是指向什么类型的指针，而不需要知道该类型内部的完整信息（大小、成员布局等）。因为所有指针的大小在同一个平台上是固定的（例如 32 位系统 4 字节，64 位系统 8 字节），所以编译器可以立即为这个指针成员分配空间。

3. **“不完全类型”可以用于指针，不能用于变量本身**  
   如果写成 `struct Node next;`（不是指针，而是结构体变量本身），编译器就必须知道 `struct Node` 的完整大小才能分配内存，但此时结构体还没定义完，大小未知，会形成“无限递归”的定义，因此这种写法是非法的。而指针只存地址，所以合法。

4. **这就是链表、树等自引用数据结构的基础**  
   通过这种“指向自身的指针”，可以把同类型的多个节点串联起来。

###### 示例
```c
/* 合法：自引用指针 */
typedef struct Node {
    int data;
    struct Node *next;  // 指针，没问题
} Node;

/* 非法：自引用结构体变量 */
struct Bad {
    int data;
    // struct Bad next;  // 错误：字段 next 是不完整的类型
};
```


#### 3. 先定义后起别名（两行清晰写法）
```c
struct Node { int data; struct Node *next; };
typedef struct Node Node;
```
第二行去掉 `typedef` → `struct Node Node;`  
变量名 `Node` 升级为 `struct Node` 的别名。

---

### 易混淆点
- `typedef struct { ... } Point;` 里的 `Point` 是**类型别名**，不是变量。
- 无论结构体是匿名还是带标签，规则不变：**花括号或 `struct 标签` 后面，原本该写变量名的地方就是别名。**


## `#define` 宏定义

### 本质
> **纯文本替换，发生在预处理阶段，完全不懂 C 语法，只是机械复制粘贴。**

### 基本形式
```c
#define 名字  替换文本
```
源码中所有独立出现的“名字”都会被直接替换为“替换文本”，然后再进入编译。

### 与 `typedef` 的关键区别

|      | `#define`                                                       | `typedef`                                            |
| ---- | --------------------------------------------------------------- | ---------------------------------------------------- |
| 处理阶段 | 预处理（编译前）                                                        | 编译阶段                                                 |
| 机制   | 文本替换                                                            | 真正的类型别名                                              |
| 作用域  | 无作用域（从定义处到文件尾）                                                  | 遵循块作用域规则                                             |
| 类型检查 | 无，直接替换                                                          | 有，编译器理解                                              |
| 指针示例 | `#define P_INT int*`<br>`P_INT a, b;` → `int* a, b;`（b 是 `int`） | `typedef int* P_INT;`<br>`P_INT a, b;` → 两个都是 `int*` |

### 带参数宏（函数宏）
```c
#define SQUARE(x)  x * x
```
**纯文本替换**，不计算参数，不关心优先级：
```c
SQUARE(5 + 1)   →   5 + 1 * 5 + 1   = 11  （错误）
```
正确做法：**把参数和整体都用括号包起来**
```c
#define SQUARE(x)  ((x) * (x))
```

### 常见用途
- 定义常量：`#define PI 3.14159`
- 条件编译：`#ifdef DEBUG`
- 简单内联“函数”：`#define MAX(a,b) ((a)>(b)?(a):(b))`

### 注意事项
- 带参数宏调用时，若参数有副作用，可能被多次求值。  
  例：`MAX(i++, j)` 会导致 `i++` 执行两次。
- 多语句宏要用 `do { ... } while(0)` 包裹，防止悬挂 else 等问题。

### 一句话
`#define` 是预处理器提供的“文本查找替换”，简单灵活但无类型安全，复杂场景优先用 `typedef`、`const`、内联函数等 C 语言本身特性。


# 条件编译（Conditional Compilation）

## 本质
> 预处理器根据宏条件，**选择性保留或丢弃代码**，在编译前决定哪些代码参与真正的编译。

## 核心指令
| 指令 | 含义 |
|------|------|
| `#ifdef MACRO` | 如果 MACRO **已定义**，保留后续代码 |
| `#ifndef MACRO` | 如果 MACRO **未定义**，保留后续代码 |
| `#if EXPR` | 如果常量表达式 EXPR 为真（非0），保留后续代码 |
| `#elif EXPR` | 相当于预处理阶段的 `else if`，前面条件不成立时再判断 |
| `#else` | 前面所有条件都不成立时，保留后续代码 |
| `#endif` | 结束一个条件编译块 |

## `#elif` 详解
- 必须位于 `#if` / `#ifdef` / `#ifndef` 和 `#endif` 之间
- 可多次出现，构成多分支选择
- 一旦某个条件成立，其后的所有 `#elif` 和 `#else` 都被忽略
- 示例：
  ```c
  #if VERSION == 1
      // 版本1代码
  #elif VERSION == 2
      // 版本2代码
  #elif defined(EXPERIMENTAL)
      // 实验性代码
  #else
      // 兜底代码
  #endif
  ```

## 典型应用场景

**1. 防止头文件重复包含**
```c
#ifndef HEADER_H
#define HEADER_H
// 头文件内容
#endif

examle:
#ifndef LED_H
#define LED_H
xxx
#endif
```

**2. 调试代码开关**
```c
#ifdef DEBUG
    printf("调试信息：x = %d\n", x);
#endif
```

**3. 跨平台适配**
```c
#if defined(_WIN32)
    // Windows 代码
#elif defined(__linux__)
    // Linux 代码
#elif defined(__APPLE__)
    // macOS 代码
#else
    #error "Unsupported platform"
#endif
```

## 注意事项
- 条件编译在**预处理阶段**完成，不消耗运行时资源
- 所有 `#if` / `#elif` 的表达式必须是**编译时可求值的常量表达式**（常由宏组成）
- `defined(MACRO)` 运算符用于在 `#if` / `#elif` 中检测宏是否定义，比 `#ifdef` 更灵活（可组合）
- 一个 `#if ... #endif` 块中，最多只有一个分支生效
