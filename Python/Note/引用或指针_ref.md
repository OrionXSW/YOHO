## Python 引用 vs C 指针（核心提炼）

### 1. Python 中没有显式的指针类型
- 变量是**对象的引用**（类似自动解引用的指针），存储的是对象的内存地址。
- 程序员**无法直接操作地址**，不能做指针算术（`p++` 等）。

### 2. 变量是“贴在对象上的标签”
```python
a = [1, 2]   # a 指向列表对象
b = a        # b 指向同一个对象
b.append(3)
print(a)     # [1,2,3]  a 也受影响
```

### 3. 赋值改变的是引用指向，而不是复制对象
```python
a = [1,2]
b = a
a = [3,4]    # a 改指向新列表，b 仍指向原列表
```

### 4. 函数参数传递：传对象引用（call by sharing）
- **对形参重新赋值** → 不影响外部变量（因为改的是局部引用的指向）
- **修改形参引用的可变对象内部** → 影响外部（内外引用同一对象）

```python
def f(lst):
    lst.append(99)   # 修改外部可见
    lst = [100]      # 只改变局部 lst 指向，外部不受影响

a = [1,2]
f(a)
print(a)   # [1,2,99]
```

### 5. 与 C 指针的对比

| 特性 | Python 引用 | C 指针 |
|------|-------------|--------|
| 显式类型 | 无，变量即引用 | `int *p;` |
| 取地址/解引用 | 无 `&` 和 `*` | `&x`, `*p` |
| 指针算术 | 不支持 | 支持 `p++` |
| 空值 | `None`（单例对象） | `NULL` |
| 修改指向 | 直接赋值 `a = other` | `p = &other` |
| 通过引用/指针改变内容 | 调用对象的方法（如 `list.append`） | `*p = new_value` 或 `p->field` |

### 6. 可变与不可变对象在引用上的表现
- **不可变对象**（int, str, tuple）：任何“修改”操作都会创建新对象，原引用自动指向新对象（其他旧引用不受影响）。
- **可变对象**（list, dict, set）：原地修改内容，所有指向该对象的引用都能看到变化。

```python
x = 10
y = x
x += 1          # x 指向新 int，y 仍指向 10
print(y)        # 10

lst1 = [1,2]
lst2 = lst1
lst1.append(3)  # 原地修改，lst2 也看到变化
print(lst2)     # [1,2,3]
```

### 7. 查看引用计数（了解底层）
```python
import sys
a = [1,2]
b = a
print(sys.getrefcount(a))  # 通常 >=2（包含临时引用）
```

## 记忆口诀
- **变量是标签，赋值是贴纸**。
- **函数传引用，改内容可见，换指向无效**。
- **不可变更新是换新对象，可变更新是原地改**。
