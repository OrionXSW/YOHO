SSH 和 HTTPS 是两种在网络通信中非常常见、但服务于不同场景的安全协议。简单来说：SSH 用于安全地远程登录和管理服务器；HTTPS 用于安全地访问网站和传输网页数据。
# 一、SSH（Secure Shell）
1. 是什么？
SSH 是一种加密的网络协议，用于在不安全的网络中建立安全的连接，最典型的用途是远程登录 Linux/Unix 服务器并执行命令。

2. 核心功能
远程终端登录：替代不安全的 Telnet、rlogin。

文件传输：通过 scp、sftp 或 rsync over SSH 安全传文件。

端口转发/隧道：把本地的网络请求通过 SSH 加密后转发到远程网络（比如翻墙、访问内网数据库）。

Git 操作：代码托管平台常用 SSH 密钥认证来推送/拉取代码。

3. 工作原理与认证方式
加密算法：使用对称加密（如 AES）保护传输内容，非对称加密（RSA、ECDSA、Ed25519）用于密钥交换和身份验证。

两种主要认证方式：

密码认证：输入远程用户的密码（简单但易受暴力破解）。
密钥对认证（推荐）：客户端持有私钥，服务器保存对应公钥，登录时无需输入密码，更安全。
首次连接警告：第一次连接服务器时会提示确认服务器的公钥指纹，防止中间人攻击。

4. 默认端口
22（TCP）

5. 安全性特点
全程加密，防止窃听和篡改。

支持密钥认证，有效抵御密码爆破。

可配合 Fail2ban 等工具阻止恶意扫描。

# 二、HTTPS（Hypertext Transfer Protocol Secure）
1. 是什么？
HTTPS 是 HTTP + TLS/SSL 的结合体。它在普通的 HTTP 和 TCP 之间插入了一层 TLS 加密层，确保浏览器与网站服务器之间的通信内容被加密，无法被中间人窃取或篡改。

2. 核心功能
网页安全浏览：银行、邮箱、社交网站必须使用。

API 接口保护：前后端分离项目中，HTTPS 保护 JSON 数据不被监听。

身份验证：通过 数字证书 向用户证明“你访问的确实是 google.com，而不是假冒网站”。

3. 工作原理（简化版 TLS 握手）
客户端（浏览器）请求建立安全连接。

服务器发送 SSL/TLS 证书（包含公钥和域名信息）。

客户端验证证书是否由受信任的 CA（证书颁发机构，如 Let's Encrypt、DigiCert）签发。

验证通过后，双方协商一个会话密钥，后续通信全部使用该对称密钥加密（速度快且安全）。

4. 默认端口
443（TCP）

5. 安全性特点
加密传输：防止运营商插广告、防止公共 WiFi 下被嗅探密码。

完整性校验：数据如果被篡改，接收方会立即发现并断开连接。

依赖证书体系（PKI）：安全性依赖于根证书库是否干净（浏览器/操作系统内置的受信任 CA 列表）。
