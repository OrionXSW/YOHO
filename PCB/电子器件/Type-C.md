type-c有几种Pin类型: 6Pin 16Pin 24Pin

VBUS是电源的正极
CC1和CC2是负责快充的，若用不上快充的话，需要串联5.1K的电阻接地(这好像是个标准，记得查下)

# Type-C类型
| 类型 | 关键引脚 | 功能 | 典型应用 | 
| :--- | :--- | :--- | :--- | 
| **6-Pin** | VBUS, GND, CC1, CC2 | **仅支持充电/供电**，无数据传输功能。支持正反插和PD快充协议。 | 电动牙刷、小风扇、单纯充电的设备。 | 
| **16-Pin (12焊盘)** | 6-Pin基础上增加D+/D- | **支持充电和USB 2.0数据传输**，无USB 3.x高速传输。同样支持PD快充等功能。 | 开发板、鼠标、键盘等仅需USB 2.0速率的设备。 | 
| **24-Pin** | 全功能引脚 | **全功能接口**，支持充电、USB 2.0、USB 3.x/4高速数据传输、视频输出（DP Alt Mode）等。 | 笔记本、手机、高端显示器等。 | 

# CC引脚
CC 是 "Configuration Channel"（配置通道）的缩写，它是Type-C接口的“大脑”，通过检测其上的电压和电阻来判断设备连接、正反插方向、传输角色以及协商供电功率。

| 设备角色 | 角色说明 | CC引脚配置 | 电阻作用 |
| :--- | :--- | :--- | :--- |
| **设备端 (Sink/受电端/UFP)** | 你的电路板、设备 | **CC1和CC2分别接一个5.1kΩ电阻下拉到地 (GND)** | 向电源端宣告自己是“需要供电的设备”，从而请求电源。若不接，C to C线材将无法识别。 |
| **电源端 (Source/供电端/DFP)** | 充电器、电脑主机 | CC引脚接上拉电阻 (Rp，阻值变化) | 通过检测下拉电阻识别设备是否已插入。通过使用不同阻值（如56kΩ, 22kΩ, 10kΩ），告知设备其最大供电能力（900mA, 1.5A, 3A等）。 |

总而言之，一个标准的供电设计可以总结为：
*   **供电端**：CC引脚通过上拉电阻Rp连接到电源。
*   **受电端**：CC引脚通过5.1kΩ下拉电阻Rd连接到地（GND）。

 对于**16-Pin**或**24-Pin**的Type-C接口，如果要使用USB 2.0数据功能，除了CC引脚的电阻，还需要注意将D+和D-数据线正确连接到您的设备上。
